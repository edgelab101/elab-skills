# 沉淀收口（所有 elab skill 的统一结尾 · SSOT）

> **纪律**：任何 elab 对话聊出**有价值的产出**（一个判断 / 一条规律苗头 / 一个交易想法）时，**结尾默认提醒用户沉淀一次**——别让想清楚的东西随对话丢掉。**提醒，不硬存**（用户说不用就算了）。

## 三档路由（按产出类型走对地方）

| 产出类型 | 存到哪 | 怎么存 |
|---|---|---|
| **想接着用的分析状态**（这次研究/诊断的中间态，下次要续） | `/elab-save` | 状态存档，`/elab-restore` 拉回来接着用 |
| **交易想法（还没验证）**（刚冒出来的念头/判断，如"IV 高位卖 put 好像稳") | `elab-trade` playbook **想法区** | 标 `[想法 待验证]`，记想法+依据+待验证什么+日期（见 `playbook-mode.md §〇`） |
| **成型规律 / 打法**（有样本+逻辑+失效条件） | `/elab-trade` Mode C | 过四关炼成正式 playbook 条目 |

## 标准收口话术（每个 skill 结尾用这套，措辞可微调）

> 这次聊出的东西值得留就别丢：
> - 想**接着往下做** → `/elab-save` 存这次状态
> - 有个**交易想法（还没验证）** → 存进 playbook 想法区（`/elab-trade`），标 `[想法 待验证]`，之后攒样本验证
> - 已经是**成型规律** → `/elab-trade` 炼成 playbook 条目
> 要我帮你存哪档？（不用也行）

## 为什么这么设计
- **想法不丢**：散户最大的浪费是"当时想到了、没记，下次重新踩坑"。想法区就是接住 raw idea 的地方。
- **不硬升级**：想法 ≠ 规律 ≠ 打法。逐级晋级（想法→规律→条目），每级有门槛（见 playbook 四关），防止把"最近赚钱的做法"当规律过拟合。
- **闭环**：沉淀不是终点——开仓前回查 playbook（含想法区待验证项）、复盘回流更新，才让沉淀变成 edge。
