# 仓位哲学分册（期权仓位怎么定）

> elab-trade 的仓位方法论。立案（Mode A）和诊断（Mode B）都引用这里。**这是教育框架、不是给你的具体数字建议**——最终比例按你自己的风险承受定，本册只给"怎么算、为什么"（930：不给方向、不给该投多少）。

## 一、为什么期权仓位不能靠直觉

期权自带杠杆：一笔看着"不大"的权利金，背后名义敞口可能是它的几十上百倍。**用付出的 premium 当仓位大小 = 系统性低估风险**。仓位的唯一正确度量是**这笔最坏能亏多少（max loss）**，不是它占用多少钱。

- 价差（defined risk）：max loss = 宽度 − 收到的 credit（可精确算）——这正是期权相对正股的优势：风险能事先钉死。
- 裸卖（undefined risk，如裸 put/裸 call）：max loss 极大甚至理论无限，**散户默认避开或按"被行权接股的全额"当风险算**。

## 二、管住两个数（整套仓位的地基）

**① 单笔风险预算**：任何一笔交易，最坏亏损 ≤ 账户的固定比例。
- 默认建议：**单笔 max loss ≤ 账户 1-2%**（保守派 1%，能承受波动的 2%；**别超 2%**）。
- 张数怎么算：`张数 = (账户 × 单笔% ) ÷ 每张 max loss`。
  - 例：账户 10 万、单笔 1% = 1000 可亏；一个 BPS 每张 max loss = (5 宽 − 1.2 credit)×100 = 380；张数 = 1000 ÷ 380 ≈ 2 张（向下取整，别凑整）。

**② 账户总风险敞口（heat）**：所有未平仓头寸的 max loss 合计 ≤ 账户的一个上限。
- 默认建议：**总 heat ≤ 账户 6-10%**（同时在场的所有单子一起爆的总亏损）。
- **相关标的要合并算**：3 笔都押半导体 = 一个风险，不是三个独立小仓。按板块/underlying 聚合看 heat。

## 三、期权特有的三个坑

1. **notional ≠ risk**：别用名义市值或 premium 当仓位度量，只用 max loss。
2. **被行权后的股票仓位要纳入 heat**：卖 put 被 assign 接了股，那笔股票的下行风险即时进总敞口，不能当"期权平了就没风险了"。
3. **到期集中度**：同一周堆一堆到期 = gamma/事件风险集中，即使每笔都合规，叠一起是一个大风险。按到期日分桶看。

## 四、机械执行（仓位规则最容易在情绪下破功）

- **进场前就把这笔的 max loss、张数、占账户% 写进 elab-trade 立案单**（写完不改）。
- 亏损、FOMO、报复交易时**最想干的就是破仓位规则**——规则事先定死，破例必须在决策日志里记一笔 `[破例 date + 为什么 + 后果自负]`，让破例**可见、可回溯**（连回内核命题：看得见的决策才可能被迭代）。
- 不确定就减小，不放大。加仓要有预先写好的加仓规则，不即兴。

## 五、和内核命题的关系

仓位不是拍脑袋，是**可算、可写下、可复盘**的决策。把仓位规则和每次是否遵守都摊进决策日志——几十笔后你能看见"我在什么情绪下最容易破仓位规则、破了之后结果如何"。**这才是仓位管理真正的 edge：不是知道 1% 这个数字，是让自己的仓位纪律可见、可迭代。**
