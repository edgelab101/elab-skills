# Mode C · playbook 沉淀（把规律炼成成型打法）

> elab-trade 的 playbook 分册。用户要"把攒下的规律炼成可复用 playbook"时走这里。数据是用户自己的，教方法不给方向（930）。

## 输入（从哪来）
- **`00_想法区/`（raw 交易想法，见 §〇）** —— 沉淀的最上游入口
- Mode A 的 `02_规律/`（随手记的规律）+ `03_定格/` 平仓复盘
- Mode B 交割单复盘出的"赚钱策略 / 高频错误"
- 用户口述"我一般怎么做 X"

**没素材别硬编**：让用户先用 Mode A 记几笔、或 Mode B 复盘一轮，攒够样本再炼。

## 〇 · 想法区（raw 交易想法暂存 · 沉淀的入口）⭐
不是所有东西都够格当 playbook 条目。刚冒出来、**还没验证**的交易想法先落这里——别丢，也别假装它已经是规律。这是"沉淀自己的交易想法"的家。

**格式（每条一小块，存 `00_想法区/想法.md` 或 playbook 顶部）**：
```
- **想法**：<一句话，如 "IV 分位>75 卖 put 收租好像稳">
- **当时依据**：<为什么这么想——某个观察/数据/哪笔交易触发的>
- **待验证什么**：<要攒什么样本 / 看到什么才能确认或推翻>
- 标签+日期：`[想法 待验证]` <date>
```

**晋级漏斗（想法是起点不是终点）**：
```
[想法 待验证] → 相关交易每次回填、攒同类样本 → 够 n 笔 → 转 02_规律 → 过四关(§一) → 成型 playbook 条目(§二)
```
- **进度可视化（让用户看到积累、别因为反馈周期长而放弃）**：每条想法/暂定规律**显式标晋级进度**——`[想法 待验证 · 已攒 3/10 笔，还差 7 笔升级]`。每次相关交易回填时 +1 并更新这个计数。**攒样本本身就是进展**，看得见的进度是留存的关键（散户特定市况一年可能就 3-5 次机会，n≥10 要跨年，没进度条会以为"白记"）。
- 被证伪的想法 → 标 `[证伪 <date>]` **留着别删**（看得到想法的生死 = 认知进化的痕迹）
- 开仓前也回查想法区的待验证项（"这次是不是在验证那个想法？"）
- **禁止**：想法直接当规律下重注（没过样本+四关 = 过拟合到直觉）。**n≥10 门槛不放宽**（n=5 胜率置信区间宽达 ±40pp）——留存靠进度可视化，不靠降标准

## 一、一条规律怎么才算数（炼法 · 命门）⭐
别把"最近赚钱的做法"当规律（容易是运气 + 过拟合到最近行情）。进 playbook 要过四关：
1. **样本量够**：标清 n=多少。**n<10 一律标 `[样本不足 待验证]`，不当正式条目执行**——散户在特定市况全年可能就 3-5 次机会，n=5 的胜率 95% 置信区间宽达 ±40pp，写成"胜率 80%"是自欺。n≥10 且有逻辑才转正式；不够就当"暂定假设"继续攒
2. **有逻辑解释**：说得清"为什么会赢"（收 theta？波动率均值回归？），不只是统计巧合
3. **分离运气 vs 技巧**：结果好≠规律对，问"同样条件再来还合理吗"
4. **配失效条件**：什么市况/前提下不灵（没失效条件的规律是假规律）
过不了的标 `[AI暂定模式 待更多样本]`，先别当正式条目。

## 二、playbook 条目框架（每条策略一张，每字段怎么填）
```
## 策略：<名字，如 IV 高位卖 Put 收租>
- 适用市况：<可判断：趋势/震荡 + IV 高低 + 事件前后>，如"IV 分位>70 且无临近财报"
- 进场条件：<量化到能照做：标的筛选/IV 分位/技术位/Delta 区间>
- 仓位规则：<单笔占比上限 + 加仓规则>，如"单笔≤总仓5%，不加仓"
- 止盈：<规则>，如"收到 50% credit 平"
- 止损/认错：<明确触发的证伪条件>，如"跌破 X 支撑/亏 Y% 平"
- 对冲：<配什么对冲>（参 elab-deconstruct 对冲易混表，别把 CC/BPS 当真对冲）
- 失效条件：<什么市况不成立>，如"IV 持续低位/单边下杀别用"
- 历史表现：<你真实数据：胜率/盈亏比/样本 n>，标 [我的数据]，n 不够标"样本不足"
```
填不出的标"待补"，**不替用户编**（尤其历史表现的胜率/数字，只用真实数据）。

## 三、沉淀流程
1 按策略类型聚类（方向性/卖方收租/价差/对冲）→ 2 每类规律过四关，够格炼成条目、不够进"暂定"区 → 3 真实数据算的标 `[我的数据]`、推断标 `[推断]` → 4 写成 `playbook.md`（用户自己的），存本地或 elab-trade 项目根

## 四、回头用闭环（别只沉淀不用）
- **开新仓前**：Mode A 立案时回查 playbook 对应条目——进场条件满足没？仓位规则守没？→ 把沉淀变纪律
- **复盘时**：Mode B 发现的新错误/规律 → 回流更新 playbook 条目
- **定期**：失效规律标 `[修正 date]`，别删（看得到进化）

## Worked example
5 笔卖 put（来自 02_规律 + 平仓复盘）→ 聚类"卖方收租"→ 过四关：逻辑=IV 高收 theta+均值回归成立、剔掉 2 笔靠大涨的运气后只剩 3 笔真符合 IV>75、失效=单边下杀被击穿——但 **n=3 远不够，标 `[样本不足 待验证]` 当暂定假设**，不写成"胜率 X%"的正式条目 → 闭环：把"IV>75 分位卖 put 收租"当**待验证方向**，之后每次同类交易回填累积，攒到 n≥10 再评估是否转正式；期间下单前查这条提醒自己 IV 没到 75 别按这套上。
