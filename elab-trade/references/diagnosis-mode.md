# Mode B · 交割单诊断（持仓结构 + 历史成交复盘）

> elab-trade 的诊断分册。用户要"诊断现有持仓 / 复盘一批交割单"时走这里。数据是用户自己的，中性诊断不给方向（930）。

## 流程
```
1 确认券商+接入 → 2 字段归一 → 3 FIFO 配对 → 4 持仓诊断（未平仓）/ 5 交割单复盘（已平仓，重头戏）
```

## 1 · 确认券商 + 接入
先问："你用哪家？富途/长桥/IBKR/其它？手动导出还是 API？" 按答案翻 `broker-ingest.md` 对应小节照做。**必提醒**：导出文件含账户号/资金额，给我前自己删，只留成交字段。

## 2 · 字段归一（标准 schema）
`trade_date | symbol | underlying | side | qty | price | fee | [expiry|strike|cp]`
**方向归一最关键**（错了全错）：买入/卖出/卖空/买入平仓、Buy/Sell+开平 → **BUY / SELL / SELL_SHORT / BUY_TO_CLOSE**。期权代码拆出 underlying/expiry/strike/cp。列名对不上让用户发表头现场映射。

## 3 · FIFO 配对算法
⚠️ 最易出错（专业脚本都踩过：同日开平误判成到期收满 credit、漏配 spread 单腿）。严格按：
1. 按合约分组（期权按完整合约 underlying+expiry+strike+cp，正股按 symbol），各自独立 FIFO
2. 组内按 trade_date 升序
3. 开仓队列（先开先平）：多头 BUY 开→入队、SELL 平→队头配；空头 SELL_SHORT 开→入队、BUY_TO_CLOSE 平→队头配
4. 每对盈亏：多头 `(平−开)×qty×乘数−fee`；空头 `(开−平)×qty×乘数−fee`；乘数=正股1/美股期权100
5. 数量不等就拆配
6. **期权到期路径（券商报表里是特殊行，不是普通成交，别漏）**：
   - 到期归零（worthless）→ 按价格 0 当 SELL（多头）/ BUY_TO_CLOSE（空头）配对，卖方吃满 credit、买方亏光
   - 长仓行权（exercise）/ 短仓被 assignment → 期权合约以**行权价**配对消失，同时在**股票账户新建对应仓位**（卖 put 被 assign=接股、卖 call 被 assign=交股）
   - 在导出字段里找 expire / exercise / assign 标记
7. 剩在队列里的（且未到期）= 当前持仓（未实现，进 §4）
8. 同日开平照真实价配，别当到期收满
9. Spread 多腿各腿分别 FIFO，再按同一组合合并看净盈亏
> FIFO 有脚本最稳（用户若有 options_review 类 FIFO 引擎可复用）。

## 4 · 持仓诊断（未平仓）
| 维度 | 怎么算 | 揭示 |
|---|---|---|
| 集中度 | 各 underlying 名义市值 ÷ 总名义（HHI） | 单点风险 |
| 净 Delta | Σ(Delta×qty×乘数)，正股=1，期权用 py_vollib/券商 | 真实方向敞口 |
| 到期分布 | 期权按到期日分桶 | 同周堆叠=事件/gamma 集中 |
| 对冲缺口 | 净 Delta 方向 vs 有无反向保护 | 裸露下行？是不是真对冲（参 elab-deconstruct） |
| 策略一致性 | 仓位结构 vs 自述 thesis | 嘴上看多、仓位却收 credit？ |

## 5 · 交割单复盘方法论（重头戏）
看的是**决策质量 + 可改进模式**，不是"赚了多少"。三层：

### 5.1 思路（用什么视角）
- 看决策质量不只看单笔输赢（好决策可能亏、坏决策靠运气赢）
- 分离运气 vs 技巧（结果好≠选择对，问"再来一次还合理吗"）
- 按策略类型分组（方向性/卖方收租/价差/对冲），别混算胜率
- 比计划 vs 执行（当时计划 vs 实际做、偏差在哪——偏差是金矿）

### 5.2 逐笔查的因素
进场理由当时成不成立（thesis 质量，非事后看涨没涨）/ 仓位合不合理 / 止盈止损按纪律没 / 持有时长 vs 计划（赢家拿太短？输家扛太久？）/ 情绪干扰（FOMO/报复/不认错）/ 对冲到位没 / 同类错误是否重复

### 5.3 该得出的结论与指导
- 哪类策略真赚钱（扣运气）、哪类一直亏 → 加重/减少哪类
- 高频错误模式（过早止盈/扛单/方向词混用）→ 对应纪律修正
- 纪律漏洞（哪条规则总不执行）→ 固化（转 Mode C playbook 沉淀）
- 可复用规律 → `02_规律/`
- 产出是"执行/选择该改什么"，**不是**"以后只买 X"（930）

## Worked example
一笔 BPS：同日 SELL_SHORT put@A（收1.20）+ BUY put@B（付0.40）；两周后 BUY_TO_CLOSE put@A（0.30）+ SELL put@B（0.05）。
1 归一四行两合约方向归一 → 2/3 FIFO：put@A 空头(1.20−0.30)×100−fee、put@B 多头(0.05−0.40)×100−fee → 合并成这笔 BPS 净盈亏（别只看 A 腿说收满 credit、别因同日误判）→ 5 复盘：进场看多理由成立吗？14 天平是按计划还是被动？属赚钱那类吗 → 中性陈述 + 方法建议，不给方向。
