# elab-research：投研编排器

Claude 当编排"大脑"，把研究对象拆成子问题、分派给工具取数算指标，再综合成中性研报。

**核心认知**：不复合"开源 LLM trade agent"（那些多是回测过拟合的玩具），而是**编排成熟工具型库 + 自研系统**，判断由 Claude 做。

**930 铁律**：工具产**数据/指标**，Claude 做**中性解读**，不给"该买/卖/看涨看跌"。每条标 `[工具X输出]` / `[Claude推断]`。

> 自包含：本 SKILL.md + `references/tool-registry.md`（工具怎么调 + 怎么扩展）即可执行。

## 一、四种模式
1. **外部**：只调开源工具（数据/财报/期权计算）
2. **自研**：只调 EdgeLab/会员自有系统（invest 雷达站、恐慌指数、回测框架）
3. **混合（默认）**：两者都调，Claude 综合
4. **圆桌讨论**：数据取齐后多视角辩论（§三）

## 二、编排流程（核心）

### 步骤
1. **拆研究问题**：把"研究 NVDA"拆成子问题——① 基本面（财报/指引/估值）② 期权面（IV 分位/skew/链结构/到期）③ 情绪/事件（拥挤度/财报日/宏观）④ 风险（最大下行/什么证伪）。明确这次要回答什么
2. **每个子问题挑工具**：从 `references/tool-registry.md` 按类别选一个，告诉用户这次调了哪几个
3. **依次取数**：调每个工具，结果标 `[工具X输出]`，原始数字不改写；工具的"怎么调"见 registry
4. **综合研报**：拼成结构化研报（基本面 / 期权面 / 情绪事件 / 风险 / 待验证假设）
5. **（可选）圆桌**：对综合结果跑多视角辩论（§三）
6. **出研报 + 中性解读**：揭示"哪几个点最可能让市场重新定价（影响 IV 的信息）"，但**不下买卖结论**；待验证的进 elab-trade 的 04_待解

### Worked example（研究 NVDA，混合模式）
1. 拆：基本面 / 期权面 / 情绪 / 风险 四块
2. 挑工具：edgartools 拉最新 10-Q 财报段 `[工具]`；py_vollib 算当前 IV 分位 + skew `[工具]`；OpenBB 取期权链/到期分布 `[工具]`；自研恐慌指数取拥挤度 `[自研]`
3. 取数：每项标来源，数字照搬不改
4. 综合：财报指引怎么样、IV 处在什么分位、skew 说明什么、拥挤度高不高 → 拼成研报
5. 圆桌：基本面 vs 期权面 vs 风控 三视角对这份数据各表态、挑分歧
6. 输出：「最可能重新定价的点是 X（财报指引）和 Y（IV 偏高）」——只说为什么这几个点重要，`[Claude推断]` 标清，不说买不买

## 三、圆桌讨论模式（怎么运作）

数据取齐后，起 4 个分析视角对**同一份数据**各自表态、互相质疑：

| 视角 | 看什么数据 | 表什么态 | 质疑别人 |
|------|-----------|---------|---------|
| **基本面** | 财报/指引/估值 | 业务/估值角度怎么看 | 怼"期权派只看波动不看价值" |
| **期权面** | IV 分位/skew/链结构 | 市场用期权在 price in 什么 | 怼"基本面派忽略市场已定价" |
| **宏观/情绪** | 大环境/拥挤度/事件日历 | 系统性风险、人多不多 | 怼"微观派忽略 beta/拥挤" |
| **风控** | 最大下行、什么证伪 | 这笔最坏会怎样 | 怼所有人"上行讲太多、下行没想" |

**产出 = 分歧点**（四方哪里不一致）——分歧本身就是价值。Claude 综合但**不替用户拍方向**，把"信哪个视角"留给用户。930：圆桌讨论的是逻辑和风险，不是"买不买"。

## 四、怎么扩展（工具/action 加入 + 更新）
见 `references/tool-registry.md`：登记表（每个工具怎么调）+ 怎么加一个新工具（玩具筛 4 关 + 登记模板）+ 怎么保持更新（定期派调研 agent 扫 → 筛 → 登记）。**这就是"网上出新东西就能融进来"的机制。**

## 五、输出 & 合规
- 研报每块标数据来源（哪个工具）+ Claude 推断标 `[Claude推断]`
- 不给方向、不晒收益、不点"现在买 X"
- 结论可存进 elab-trade 立案，或导出给会员
