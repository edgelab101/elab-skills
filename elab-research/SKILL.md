# elab-research：投研编排器

Claude 当编排"大脑"，把研究对象拆成子问题、分派给工具取数算指标，再综合成中性研报。

**核心认知**：不复合"开源 LLM trade agent"（那些多是回测过拟合的玩具），而是**编排成熟工具型库 + 自研系统**，判断由 Claude 做。

**930 铁律**：工具产**数据/指标**，Claude 做**中性解读**，不给"该买/卖/看涨看跌"。每条标 `[工具X输出]` / `[Claude推断]`。

**AI 边界（什么时候信 AI 分析、什么时候 AI 在误导你）**：
- ✅ 信：AI 做**结构化取数 + 客观计算**（拉财报段、算 IV 分位、跨源汇总）——这是它的强项，标 `[工具X输出]` 可核。
- ⚠️ 存疑：AI 做**因果/预测/"所以会涨跌"** ——它擅长把话说得像模像样（plausible），不等于对。标 `[Claude推断]`，且必过 §三 强制证伪。
- ❌ 别信：AI **编数字/凑分歧/给方向**（幻觉财报数、演圆桌制造伪共识、暗示买卖）——这些是 AI 最危险的地方，本 skill 用"数字照搬不改 + 强制证伪 + 不给方向"三道闸挡住。
- 一句话：**AI 是判断的放大器，不是判断本身**——它让你的分析更快更全，但对不对要你自己核，方向要你自己下。

## 一、模式（怎么选，别临时拍）
1. **外部**：只调开源工具（数据/财报/期权计算）——**用于**：标的无 EdgeLab 雷达覆盖（如冷门票/非美股），或用户只要公开数据
2. **自研**：只调 EdgeLab 雷达/恐慌/回测——**用于**：标的在雷达 watchlist 内、只想要 EdgeLab 独有市场数据/指标
3. **混合（默认）**：两者都调，Claude 综合——**用于**：标的既有公开数据又在雷达覆盖内（多数情况）
4. **多 agent 辩论（重·可选加深）**：综合完，再起**真独立子 agent** 做多空辩论 + 独立裁判（§三+）——**用于**：高不确定、大仓位、想把分歧摊透的深度研究。费 token，不是每次都上
> 决策规则：标的在雷达覆盖内 → 混合；不在 → 外部；用户明说"只要你们自己的数据" → 自研；用户要"辩论/多角度吵/深度"→ 叠加模式 4。拿不准问一句。
> 不论哪个模式，**§三 的强制自我证伪都必跑**（不是可选）；模式 4 是在它之上再加重。

## 二、编排流程（核心）

### 步骤
1. **先判研究对象：标的 or 板块**
   - **标的（个股/ETF）**：拆成子问题——① 基本面（财报/指引/估值）② 期权面（IV 分位/skew/链结构/到期）③ 情绪/事件（拥挤度/财报日/宏观）④ 风险（最大下行/什么证伪）。
   - **板块/赛道/产业链**（半导体/AI 算力链/中概等）→ **走 `references/sector-crowding.md` 拥挤度 5 维框架**（涨幅权重/估值分位/资金集中/共识热度/杠杆信号 → 拥挤度评分 + 板块级判断 + 成分股下钻），不是逐个拆基本面。板块级更合规（升板块降个股）。
2. **每个子问题挑工具**：从 `references/tool-registry.md` 按类别选一个，告诉用户这次调了哪几个
3. **取数（并发 + 超时降级，别串行卡死）**：并发调所有选中的工具，结果标 `[工具X输出]`，原始数字不改写。**任一工具超 ~30s 未返回或报错→标 `[超时/失败跳过]` 继续**；自研 API 401/503 按 registry 优雅降级。**至少 2 个工具成功才综合**，否则告诉用户"取数不足，建议重跑"。工具"怎么调"见 registry
4. **综合研报**：拼成结构化研报（基本面 / 期权面 / 情绪事件 / 风险 / 待验证假设）
5. **强制自我证伪（必跑，非可选）**：见 §三——综合后必须回答"这份研报哪个推断最可能错、依据哪个数据点最薄弱"
6. **出研报 + 中性解读**：揭示"哪几个点最可能让市场重新定价（影响 IV 的信息）"；给一段**历史分布陈述**（见下）；**不下买卖结论**；待验证的进 elab-trade 的 04_待解。**默认产出 markdown 研报**；用户要 HTML/网页版/出图分享 → 见 `references/html-report.md`（可选，自包含单文件 + 自动图）

**历史分布陈述（合规但有决策价值，替代空洞的"不给结论"）**：当拿到信号组合（如 IV 分位 X + 拥挤度 Y + 距财报 Z 天），用自研/回测的历史数据答一句"**历史上类似配置后 N 天，realized vol / 标的价格区间的分布是……**"。这是**数据性陈述不是方向建议**（描述过去分布，不预测、不推荐），但给用户真实决策参考。数据不足则说"样本不足，无法给分布"，不编。

### Worked example（研究 NVDA，混合模式）
1. 拆：基本面 / 期权面 / 情绪 / 风险 四块
2. 挑工具：edgartools 拉最新 10-Q 财报段 `[工具]`；py_vollib 算当前 IV 分位 + skew `[工具]`；OpenBB 取期权链/到期分布 `[工具]`；自研恐慌指数取拥挤度 `[自研]`
3. 取数：每项标来源，数字照搬不改
4. 综合：财报指引怎么样、IV 处在什么分位、skew 说明什么、拥挤度高不高 → 拼成研报
5. 强制证伪：这份研报最弱一环是"IV 偏高"这个判断——依据只有单日 IV 分位一个数据点，且没核实是不是财报临近的正常抬升；若这点错，整个"期权贵"结论翻
6. 输出：「最可能重新定价的点是 X（财报指引）和 Y（IV 偏高）」+ 历史分布「IV 分位 >75 且距财报<5 天的历史案例，后 10 天 realized vol 分布约…」——只说重要性和历史分布，`[Claude推断]` 标清，不说买不买

## 三、强制自我证伪（每份研报必跑，替代"假圆桌"）

> 为什么不是"多视角圆桌"：同一个 Claude 演 4 个角色只会**制造伪分歧**——它会自动平衡矛盾、把结论拉伸去"显得有分歧"，置信度反而更低，没有新信息产生。真正有用的是**逼自己找研报里最可能错的地方**。

综合研报后，**固定回答**（不许跳过）：

1. **最弱一环**：这份研报哪个推断/结论最可能是错的？
2. **最薄弱数据点**：它依赖的哪个数据点最经不起推敲（单一来源？滞后？工具估算有误差？没交叉验证？）？
3. **若它错，结论怎么翻**：这一环塌了，整份研报的方向会怎么变？
4. **反方最强论据**：跟本研报主结论相反的一方，最强的论据是什么？（不是为了辩赢，是确认自己没漏掉）

用这 4 问过一遍**基本面 / 期权面 / 情绪拥挤 / 风控下行**四个角度（当检查清单，不是当辩论角色），哪个角度的推断最薄弱就点哪个。

产出 = **诚实的置信度标注**（哪些结论稳、哪个最可能被推翻），不替用户拍方向。930：证伪的是自己的逻辑和数据，不是"买不买"。

## 三+、多 agent 辩论模式（模式 4 · 可选加深 · 真独立子 agent）

> 为什么这个是真的、圆桌是假的：**用 Task tool 起独立子 agent**——每个新 context、拿不到彼此的推理，**不会自我平衡制造伪分歧**。这才是真对抗（借鉴 FinDebate 的 Leader+Judge）。费 token，只在高不确定/大仓位/用户要"深度吵一遍"时上。

综合研报（§二 step 4）出来后，起 **3 个独立子 agent**（每个都要求"必须加载 elab-research + 用登记表工具取数、数字照搬、标来源"）：

1. **🐂 看多 agent**：只找这个标的/thesis **最强的看多论据 + 支撑数据**，别管看空。产出：看多逻辑链 + 依赖的关键数据点。
2. **🐻 看空 agent**：只找**最强看空论据 + 最大风险 + 证伪点**，别管看多。产出：看空逻辑链 + 关键风险数据。
3. **⚖️ 裁判 agent（独立·不参与前两个的生成）**：读多空两份，**只做三件事**——① 评每一方每个结论的**置信度**（依据强不强、数据硬不硬）② 指出双方**真正分歧在哪**（不是措辞差异，是事实/假设的分歧）③ 点出**哪方证据更薄弱**。**不站队、不给方向、不下"该买该卖"**。

主 agent（orchestrator）收 3 份 → 产出：
- **分歧地图**：多空在哪几点真吵、各自最硬和最软的证据
- **置信度标注**：哪些结论两方都认（较稳）、哪个最可能被推翻
- **不替用户拍方向**——把"信哪边"留给用户

**930**：辩论的是**逻辑/证据/风险**，产出是**分歧 + 置信度**，不是"买不买"。子 agent prompt 里写死"不给买卖方向"。
**降级**：Task tool 不可用 / 用户不想费 token → 回退 §三 单轮自我证伪（默认够用）。

**⚠️ 首次使用提醒（跑辩论模式前先说一句，别默默开跑）**：
- **费 token**：3 个独立子 agent 各自取数+推理，比普通研究贵不少——先问"要不要上辩论（费 token），还是普通研究够了"。
- **不需要 DeepSeek/外部 LLM key**：本模式跑在 Claude 自己身上（子 agent），**和 TradingAgents 那类需录 DeepSeek key 的本地框架无关**，别让用户去配 LLM key。
- **数据源凭据按需**：子 agent 若要用 finnhub/长桥 等需 key 的源，**首次要用户先配好自己的 key/凭据**（env/文件，见 tool-registry 凭据纪律）；没配就用免费源（yfinance/stooq）跑，别卡住。
- 首次提示模板："辩论模式会起 3 个子 agent（费 token）、跑在 Claude 上不用额外 LLM key；要更准的实时数据可先配 finnhub/长桥 凭据，没有就用免费源。开跑吗？"

## 四、怎么扩展（工具/action 加入 + 更新）
见 `references/tool-registry.md`：登记表（每个工具怎么调）+ 怎么加一个新工具（玩具筛 4 关 + 登记模板）+ 怎么保持更新（定期派调研 agent 扫 → 筛 → 登记）。**这就是"网上出新东西就能融进来"的机制。**

## 五、输出 & 合规
- 研报每块标数据来源（哪个工具）+ Claude 推断标 `[Claude推断]`
- 不给方向、不晒收益、不点"现在买 X"

## 六、沉淀收口（结尾默认提醒一次 · 详 `_shared/capture-closing.md`）
研报做完，问用户这次产出要不要留（提醒不硬存）：
- **想接着深挖/续研究** → `/elab-save` 存状态
- **研究里冒出的交易想法（没验证）** → playbook `§〇 想法区`（`/elab-trade`），标 `[想法 待验证]`，之后攒样本
- **已成型的规律/打法** → `/elab-trade` 炼成 playbook 条目
- （对外发星球 → `/elab-report` 脱敏）

### 研报 → 一笔交易（handoff 到 elab-trade 立案）⭐
用户要把这份研报变成真交易 → 直接喂 `/elab-trade` 决策立案，字段一一对上（别让研报和决策脱节）：
- 研报**主结论** → 立案的 **thesis**（标 `[本人判断]`，研报是 `[Claude推断]`，别混）
- 研报**最可能重新定价的点 + 最弱一环** → 立案的 **证伪条件 / 监控信号**（研报替你找出"哪里最容易错"，正好当认错触发）
- 研报的 **IV 分位 / 价位 / 关键数据** → 立案的进场参考（数字照搬，标来源）
- 研报**没给方向**，立案的方向/仓位 **由用户自己定**（930 不变）
