# 板块/赛道研究 · 拥挤度框架（elab-research 板块模式）

> 研究对象是**产业链 / 板块 / 主题赛道**（如"美股半导体""AI 算力链""中概"）时走这套——**板块级判断 + 成分股下钻**。
> 930：只描述**机制 / 事实 / 结构**，不给"该不该买/减/进"的操作方向。板块级本就比个股安全（合规方向 = 升板块、降个股）。

## 何时用
一条产业链/板块被市场集中热炒，要判断"挤到什么程度、结构怎样、谁在贡献拥挤"。典型触发：龙头连续新高、被编成顺口溜、杠杆 ETF 密集申请/上市、卖方集体唱多、社媒刷屏。

## 五维数据（每维取数走 `tool-registry.md`）
| 维度 | 看什么 | 默认取数 |
|---|---|---|
| **D1 涨幅+权重** | 成分股 YTD 涨幅、指数权重排名、市值集中度 | yfinance / 长桥 + 指数成分 |
| **D2 估值分位** | 当前 PE/PB vs 行业历史中枢（历史分位 %） | edgartools/finnhub + 历史 PE band |
| **D3 资金集中度** | 成交额占比、ETF 资金流、（北向若适用） | finnhub / ETF flow / 成交 |
| **D4 共识热度** | X/reddit 提及量、卖方研报频次、媒体覆盖 | 自研雷达（会员：X/资讯聚合）/ 公开搜索 |
| **D5 杠杆工具信号** | 杠杆 ETF 申请/上市、衍生品上新 | SEC EDGAR 事件（事件型，半自动/人工） |
> 缺数据的维度**按权重重分配、标 partial，不硬填**（数据真实性）。

## 拥挤度评分（Crowding Score · 0-100 · 可机读）
```
score = 0.25·涨幅分位 + 0.25·估值分位 + 0.20·资金集中度 + 0.20·共识热度 + 0.10·杠杆信号
```
- 各维归一到 0-100（分位 / Z-score）。
- 分档：**< 40 不拥挤 · 40-70 偏热 · 70-85 高拥挤 · > 85 极端拥挤**。
- 粒度：**板块级 score + 下钻成分股级**（谁贡献了拥挤、谁在分化）。

## 五步 SOP（照此跑，不重想）
1. **D1 涨幅权重**：这条链涨了多少、龙头是否挤进指数权重前排
2. **D2 估值分位**：当前估值是历史中枢几倍 / 处历史几分位（业绩能撑多少）
3. **D3 资金集中**：成交/ETF/资金是否高度集中在头部几只
4. **D4 共识普及**：社媒/卖方/媒体——认知是否已全民普及（筹码满的旁证）
5. **D5 杠杆信号**：杠杆工具密集上市 = 赛道后半程旁证（**非择时指标，只是一个侧面**）
→ 综合拥挤度评分 + 监控信号 → **必过 §三 强制自我证伪**（这个板块判断最弱一环在哪）

## 结论结构（930）
拥挤度**档位** + **五维拆解** + **监控信号**（成交萎缩 / 资金转向 / 估值见顶）+ **工具机制描述**。
- 只描述机制/结构/事实，**0 操作建议、0 方向、0 时机判断**。
- 落点是"拥挤度信号本身"，不带"所以该减/该等"的时机判断。

## 和其他模式的关系
- **成分股下钻** = 转标的研究（主 SKILL §二）那套（对龙头单独跑）。
- **可叠加多 agent 辩论（模式 4）**：让多空两方对"这个板块拥挤度/还能不能上"吵一遍 + 独立裁判评置信度。
- **真实拥挤度雷达数据**（实时 score / 历史序列）走会员自研 API（token gated）——本框架是**方法（免费）**，数据是**会员权益**。
